Analytics

Fastback's analytics helpers are small diagnostics built on top of recorded trades and collected account values. They do not change execution, accounting, commissions, or settlement behavior.

Use them for quick checks after a backtest:

  • performance summaries from return or equity series
  • quote-currency realized P&L diagnostics from Trade records
  • compact trade summaries
  • FIFO holding-period reconstruction
  • realized P&L concentration by instrument, period, or trade

Performance Summary

performance_summary summarizes a periodic return series, or a PeriodicValues equity collector. It returns a PerformanceSummary with compact REPL printing.

using Fastback
using Dates
using DataFrames

performance_summary([0.01, -0.004, 0.007, 0.002]; periods_per_year=252)
PerformanceSummary(
    tot_ret=0.015027723439999807,
    cagr=1.5592220583380496,
    sharpe=9.710327781899489,
    sortino=29.764702249476645,
    calmar=389.80551458450816,
    max_dd=0.0040000000000000435,
    avg_dd=0.0010000000000000109,
    ulcer=0.0020000000000000217,
    vol=0.09731906288081488,
    n_trades=0,
    n_closing_trades=0,
    winners=missing,
    losers=missing
)

When you pass an account, the summary also reports executed trade count, closing trade count, and win/loss ratios from recorded closing trades:

performance_summary(acc, equity_data; periods_per_year=252)

Use performance_summary_table when you want the same fields as a one-row Tables.jl-compatible table. Convert it to a DataFrame when that is convenient:

DataFrame(performance_summary_table(equity_data; periods_per_year=252))

For an equity collector:

collect_equity, equity_data = periodic_collector(Float64, Day(1))
collect_equity(DateTime(2026, 1, 1), 10_000.0)
collect_equity(DateTime(2026, 1, 2), 10_100.0)
collect_equity(DateTime(2026, 1, 3), 10_050.0)

performance_summary(equity_data; periods_per_year=252)

Trade Diagnostics

The trade diagnostics operate on acc.trades or on any vector of Trades. trade_summary groups monetary fields by currency: quote-currency summaries carry return diagnostics, while settlement-currency summaries carry settlement cash P&L and commissions.

acc = Account(;
    funding=AccountFunding.Margined,
    base_currency=CashSpec(:USD),
    broker=FlatFeeBroker(fixed=1.0),
)
deposit!(acc, :USD, 10_000.0)

inst = register_instrument!(acc, spot_instrument(Symbol("ABC/USD"), :ABC, :USD))

dt0 = DateTime(2026, 1, 1)
fill_order!(
    acc,
    Order(oid!(acc), inst, dt0, 100.0, 2.0);
    dt=dt0,
    fill_price=100.0,
    bid=100.0,
    ask=100.0,
    last=100.0,
)

win_trade = fill_order!(
    acc,
    Order(oid!(acc), inst, dt0 + Day(1), 110.0, -1.0);
    dt=dt0 + Day(1),
    fill_price=110.0,
    bid=110.0,
    ask=110.0,
    last=110.0,
)

loss_trade = fill_order!(
    acc,
    Order(oid!(acc), inst, dt0 + Day(2), 90.0, -1.0);
    dt=dt0 + Day(2),
    fill_price=90.0,
    bid=90.0,
    ask=90.0,
    last=90.0,
)

trade_summary(acc)
TradeSummary(
    trades=3,
    realized=2,
    hit_rate=0.5,
    quote_summaries=QuoteTradeSummary[QuoteTradeSummary(
    symbol=USD,
    trades=3,
    realized=2,
    commission=3.0,
    gross_pnl=0.0,
    net_pnl=-3.0,
    realized_notional=200.0,
    gross_return=0.0,
    net_return=-0.015,
    hit_rate=0.5,
    avg_win=8.5,
    avg_loss=-11.5,
    payoff_asymmetry=0.7391304347826086
)],
    settlement_summaries=SettlementTradeSummary[SettlementTradeSummary(
    symbol=USD,
    trades=3,
    realized=2,
    commission=3.0,
    gross_pnl=0.0
)],
    avg_holding=129600000 milliseconds
)

The quote-currency P&L helpers are direct companions to the realized return helpers:

(
    gross_pnl_quote=gross_realized_pnl_quote(win_trade),
    net_pnl_quote=net_realized_pnl_quote(win_trade),
    net_return=realized_return_net(win_trade),
)
(gross_pnl_quote = 10.0, net_pnl_quote = 8.5, net_return = 0.085)

These helpers report quote-currency P&L. They are useful for trade diagnostics, but they are not a replacement for t.fill_pnl_settle when settlement currency or point-in-time FX conversion matters.

Holding Periods

realized_holding_periods reconstructs FIFO lots by instrument symbol from the trade stream. Ordinary open/close trades and partial exits are exact under FIFO. Scale-in, reduce, and flip cases are FIFO approximations because Fastback stores one netted position per instrument, not full lot identity.

realized_holding_periods(acc)
2-element Vector{RealizedHoldingPeriod{Dates.DateTime, Dates.Millisecond}}:
 RealizedHoldingPeriod(
    symbol=ABC/USD,
    entry=2026-01-01T00:00:00,
    exit=2026-01-02T00:00:00,
    quantity=1.0,
    period=86400000 milliseconds
)
 RealizedHoldingPeriod(
    symbol=ABC/USD,
    entry=2026-01-01T00:00:00,
    exit=2026-01-03T00:00:00,
    quantity=1.0,
    period=172800000 milliseconds
)
holding_period_summary(acc)
HoldingPeriodSummary(
    lots=2,
    quantity=2.0,
    avg=129600000 milliseconds,
    median=86400000 milliseconds
)

If a trade vector starts after a position is already open, unmatched realized quantity is skipped because the entry timestamp is not present in the trade stream.

Supported by values are :instrument, :period, and :trade. Supported period buckets are :day, :month, and :year. Period grouping requires date-bearing timestamps; Dates.Time-only trade streams can use :instrument or :trade grouping.
